# Decisions

## D-001 ストレージ設計
- 日付：2026-04-09
- 決定：GitHub リポジトリ + Vercel サーバーレス関数中継
- 理由：GistはスキーマGistはスキーマ管理が弱い。Vercel KV より Git 管理の使い勝手が優れる。PATはVercel環境変数に隠してセキュリティを担保。

## D-002 検索・取得方針
- 日付：2026-04-09
- 決定：対応サイト2〜3件のパーサー + 未対応サイトはAI抽出のハイブリッド
- 理由：リアルタイムスクレイピングはレイテンシ・bot対策リスクが高い。一括取得＋ローカルインデックスで速度を確保。

## D-003 補正プリセット設計
- 日付：2026-04-09
- 決定：URL自動判定でプリセット自動選択。フォントズレ警告は実装しない。
- 理由：警告はUXを損なう。自動適用のみで十分。

## D-004 開発優先順序
- 日付：2026-04-09
- 決定：Phase 1（Vercel API + GitHub同期）→ Phase 2（PWA）→ Phase 3（Chrome拡張）
- 理由：共通APIを先に固めることでPWA・拡張がUIに集中できる。
